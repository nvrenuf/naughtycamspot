# Model Hub + Offer Structure (Pinned Note)

This note explains the front-door logic for NaughtyCamSpot and how that logic connects to the broader offer ladder.

It should be read alongside:
- `docs/NCS_POSITIONING_AND_OFFER_LADDER.md`
- `docs/MODEL_DISCOVERY_NEW_LANE_PIN.md`
- `docs/MODEL_REVENUE_ARCHITECTURE_PIN.md`

## Core decision

NaughtyCamSpot should control the **front door experience** when NCS is being held accountable for outcomes.

At the same time, the model should retain meaningful ownership where it matters:
- her accounts
- her content
- her identity assets
- her custom domain when she is paying for a standalone premium setup

## Why the hub matters

The hub is not a cosmetic extra.
It is the place where discovery becomes direction.

Its job is to:
- organize first impressions
- create a cleaner path into the model's world
- route fans toward the right current CTA
- support event and VIP logic
- reduce reliance on random platform behavior

Without a controlled front door, NCS can influence traffic but cannot fully shape conversion.

## Product split: front door ownership and control

### Standard

**NCS-hosted managed hub**

- hosted as a NaughtyCamSpot page such as `/model-name`
- managed by NCS while service is active
- NCS controls layout, CTA order, routing, analytics, event blocks, and optimization
- positioned as a polished managed creator front door inside the NCS system

Important framing:
- do not present this as a cheap fallback
- do not treat it like a dressed-up link page
- present it as the standard managed hub inside the NCS operating system

### Premium

**Model-owned custom domain + NCS-managed hub**

- the model owns the domain
- NCS builds and manages the hub experience on that domain while service is active
- this is the stronger standalone-brand option
- if service ends, the model keeps the domain and can repoint it elsewhere

Important framing:
- if the model wants a true standalone creator brand, premium is the right fit
- the premium solution should not be a visible redirect to NaughtyCamSpot

## Explicit non-goal

Do **not** present a model-owned domain that visibly resolves to a NaughtyCamSpot path as the premium solution.

That weakens:
- premium feel
- standalone brand credibility
- the model's sense of ownership

## Third-party link tools

Linktree, Beacons, and similar tools are not the core architecture.

Working position:
- the model can use them if she wants
- they may exist as side tools or backups
- they should not define the premium NCS system

## Accountability principle

If NCS is responsible for outcomes, NCS should control:
- hub layout
- CTA order
- routing priorities
- featured events and campaigns
- VIP capture placement
- testing and optimization decisions

This applies to both standard and premium while the package is active.

## Ownership principle

Trust posture must remain intact:
- no passwords
- no exclusivity
- model owns accounts and content
- cancel anytime
- no fake "official partner" framing
- no spam DM automation

Premium adds one more ownership rule:
- the model owns the domain and keeps it if service ends

## Offer ladder relationship

The hub logic should map cleanly to the broader offer ladder.

### 1. Foundation / Start Right

Purpose:
- help a model avoid bad starting decisions
- reduce early identity mistakes
- create basic clarity before growth work

Strategic role:
- naming and positioning guidance
- account/pathway clarity
- baseline launch readiness
- early trust posture and ownership clarity

At this stage, the model may not need a full custom hub yet.
The focus is getting the brand and starting direction right.

### 2. Launch

Purpose:
- give the model a cleaner public starting point
- establish the first real front door
- make discovery efforts point somewhere coherent

Strategic role:
- initial managed hub setup
- basic routing priorities
- event/schedule presence
- first discovery-to-destination structure

For many models, the standard NCS-hosted hub is sufficient here.

### 3. Brand Builder

Purpose:
- strengthen the model's identity as a real brand, not just a stream account
- improve consistency across public rails, hub, and offers

Strategic role:
- stronger creative direction
- cleaner persona and event language
- more deliberate front-door experience
- possible move to premium custom-domain structure when justified

This is where the hub becomes more than a utility page.
It becomes brand infrastructure.

### 4. Growth Partner

Purpose:
- support a model as an active business with recurring optimization
- treat the hub as part of a broader system, not a one-time build

Strategic role:
- ongoing routing and CTA testing
- event-led campaigns
- VIP capture and retention logic
- premium offer positioning
- deeper revenue-architecture support

At this stage, the hub is one operating layer inside a larger model business system.

## Supporting premium add-ons

This structure fits well with higher-value identity and infrastructure support such as:
- model-name availability scan across platforms
- social handle scan
- domain-name scan
- identity recommendations
- custom hub build
- branded routing hierarchy

## Forward-looking product note

Today, the hub is part of a service-led model.

Long term, this can become part of a broader operator system where:
- models are the direct user/beneficiary
- agencies, promoters, studios, and operators may become the buyer/operator layer
- the hub is one managed surface inside a larger B2B/operator workflow

That is a strategy direction, not a promise of a near-term software product.

## Operator-delivery note

The hub/front-door logic does not change just because the commercial relationship changes.

Whether NCS works:
- directly with a model
- or through an agency, studio, promoter, or operator

the same rule still applies:
- if NCS is accountable for outcomes, NCS should control the front door while the service is active

The beneficiary may still be the model even when the buyer is an operator layer above her.

## Clean summary

**Standard:** NCS-hosted managed hub on NaughtyCamSpot

**Premium:** model-owned custom domain with NCS-managed hub

The hub is the controlled front door.
The offer ladder determines how much brand, routing, and revenue-system sophistication is built around it.
