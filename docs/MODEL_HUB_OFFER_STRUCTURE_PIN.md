# Model Hub Offer Structure — Pinned Note

This note captures the current decision on model hub ownership, hosting, and package structure.

## Core decision

NaughtyCamSpot should control the **front door** experience if NCS is being held accountable for results.

At the same time, the model should retain meaningful ownership of her identity assets where appropriate.

## Product split

### Standard package
**NCS Hosted Hub**
- hosted as a NaughtyCamSpot page such as `/model-name`
- managed by NCS
- NCS controls layout, CTA order, routing, analytics, event blocks, and hub optimization while the package is active
- positioned as a strong, polished, premium hosted creator front door inside the NCS system

Important framing:
- do not present this as a cheap fallback
- present it as a managed model hub / hosted creator front door

### Premium package
**Model-Owned Custom Domain + NCS-Managed Hub**
- the model owns the domain
- NCS builds and manages the hub experience behind that domain while the package is active
- this is the true standalone brand option
- stronger for long-term creator identity and premium positioning

Important framing:
- if the model wants a real standalone brand, premium is the correct package
- the public-facing domain should remain the model’s own domain, not a visible redirect to NaughtyCamSpot

## Explicit non-goal

Do **not** point a model-owned custom domain at a visible NaughtyCamSpot path as the premium solution.

Reason:
- it weakens the premium feel
- it reduces the sense of independent creator brand
- it makes premium look like a dressed-up redirect

## Third-party link tools

Services such as Beacons / Linktree / similar are not the core architecture.

Working position:
- if the model wants to manage a third-party link page herself, that is her choice
- it is not the centerpiece of the NCS premium offering
- it can exist as a side tool or backup, but it should not drive the main system design

## Why this structure works

It balances:
- NCS control over the conversion environment
- model clarity about ownership
- stronger premium positioning
- a clear standard vs premium ladder
- less confusion about who manages what

## Commercial positioning

Recommended public framing:

**Standard:** a great NaughtyCamSpot managed model hub
**Premium:** a custom-domain creator brand hub managed by NCS

This keeps the offer simple:
- if you want a strong managed front door, NCS hosted hub is enough
- if you want a true standalone creator brand, premium custom domain is the answer

## Accountability principle

If NCS is responsible for outcomes, NCS should control:
- hub layout
- CTA order
- routing priorities
- featured events / campaigns
- VIP capture placement
- testing and optimization decisions

This applies to both standard and premium while the package is active.

## Ownership principle

Premium package rule:
- the model should own the domain
- NCS should manage the hub experience during active service
- if service ends, the model keeps the domain and can repoint it elsewhere

## Supporting premium add-ons

This structure fits well with a premium creator identity package that may include:
- model-name availability scan across cam platforms
- social handle availability scan
- domain-name scan
- creator identity recommendations
- custom hub build
- branded routing / platform hierarchy

## Clean summary

**Standard package:** NCS-managed `/model-name` hub on NaughtyCamSpot

**Premium package:** model-owned custom domain with NCS-managed custom hub

This is the current recommended structure.
